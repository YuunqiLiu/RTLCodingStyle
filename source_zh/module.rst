模块
===============

模块名和文件名必须相同
-------------------------------------------------

- **每次定义新模块时，必须创建一个同名的新文件。**

这样，我们可以直接通过文件名知道文件的内容。

同时，可以通过检查文件列表中是否有重名文件来直接检查是否有重复的模块。

- **文件后缀必须与实际使用的语法对应。**

例如，如果文件中使用了 system verilog 语法，那么后缀为 v 是错误的，必须是 sv。

这是因为一些编译器通过后缀名来决定代码解析规则。

由于 verilog 是 system verilog 的子集，sv 始终是适用的。

模块定义
--------------------

以下是一个模块定义的示例。

.. code-block:: systemverilog 

    module sc_cmn_vr_dec
        // 导入包
        import scc_param_pkg::*;
    #(
        // 参数定义
        parameter integer unsigned A = 1
    )(
        // IO 定义
        input         clk    ,
        output logic  data
    );

        // RTL 代码
        // ...

    endmodule

模块定义包含四个部分。

1. **导入的包**
   
紧接在 module 之后导入的包，其作用域限定在模块内部（不包括子模块）。

2. **参数定义**
3. **IO 定义**
4. **内部电路描述**

为了统一，请尽量按照上述模板进行换行，即括号单独占一行。
